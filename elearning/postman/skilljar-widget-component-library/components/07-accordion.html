Start by establishing a transport-level connection to your MCP server. The client sends an initialize request with its protocol version and capabilities.

$ npx @modelcontextprotocol/client connect stdio ./server.js
💡 The server responds with its own capabilities, completing the handshake.

Once connected, enumerate what the server exposes. Use the three discovery endpoints:

  • tools/list — all invokable functions
  • resources/list — all readable data sources
  • prompts/list — all reusable prompt templates

Register the tool definitions with your AI model. Pass them in the tools parameter of your API call.

const { tools } = await client.listTools();
const response = await claude.messages.create({ tools, ... });
⚠️ Only include tools the model actually needs for the task — fewer tools means faster, more accurate responses.

Send a message, handle tool use responses in a loop:

  1. Send message to Claude with tools attached
  2. If response has tool_use blocks, call tools/call via MCP
  3. Append tool results to the conversation
  4. Re-send to Claude until you get a text response

After each tool invocation, validate the response before continuing:

  • Check isError: false in the tool result
  • Validate content type matches expected schema
  • Log the exchange for debugging
  • Close the connection cleanly when done
