Data

Resources

Resources expose data that the AI can read — files, database records, API responses, or any structured information. They are read-only by nature: the server defines what data is accessible, and the AI retrieves it on demand.

Analogy

"Like a library shelf — the AI can browse and retrieve, but cannot modify."

File contents DB records API data Config files 
// Resource definition
{
  "uri": "file://docs/readme.md",
  "name": "Project README",
  "description": "Main project documentation",
  "mimeType": "text/markdown",
  "contents": {
    "text": "# My Project\n\nWelcome..."
  }
}
🔧
Actions

Tools

Tools are functions the AI can invoke to perform actions — search, compute, write, send. Unlike resources, tools have side effects. The server implements the logic; the AI decides when to call them based on the task at hand.

Analogy

"Like a Swiss Army knife — each tool is a specific capability the AI can reach for."

Web search Code execution File write API calls 
// Tool definition
{
  "name": "search_web",
  "description": "Search the web for current info",
  "inputSchema": {
    "type": "object",
    "properties": {
      "query": {
        "type": "string"
      }
    },
    "required": ["query"]
  }
}
💬
Templates

Prompts

Prompts are reusable instruction templates that shape how the AI behaves for specific tasks. They can include arguments, example messages, and system context — making consistent behavior easy to reproduce across different sessions.

Analogy

"Like a job description — setting context, tone, and responsibilities upfront."

System prompts Few-shot examples Role definitions Task instructions 
// Prompt template
{
  "name": "code_reviewer",
  "description": "Review code for quality",
  "arguments": [{
    "name": "language",
    "required": true
  }],
  "messages": [{
    "role": "system",
    "content": "You are a {{language}} expert..."
  }]
}
